# <p align="center">BiliShelf</p>

<p align="center"><em style="font-size:0.2rem;">引言，我想说的话：</em></p>
<p align="center">
你是否会因为 b 站收藏有数量限制，亦或者是搜索不好用而感到烦恼？我不知道有多少人和我一样，有这些问题：根据记忆里的关键词，检索不到曾经看过的视频，以及在收藏夹上限后，他们迫不得已转向其他方式记录：web 端或是改成浏览器收藏夹记录，app 端或是改用截图保存，再有甚者忍痛割爱删除曾经的收藏夹……

- 一个面向 B 站收藏管理的本地优先浏览器扩展。
- 把视频信息、收藏关系与标签保存在本地，解决站内收藏夹数量限制和检索体验问题，提供比 B 站原生收藏更灵活的整理、检索、同步与备份能力。

## 项目定位

BiliShelf 主要解决这些常见问题：

- B 站原生收藏夹数量、层级和管理体验有限
- 想按标题、UP 主、简介、标签、日期范围更细致地搜索视频
- 一个视频可能同时属于多个整理维度，需要跨收藏夹管理
- 担心视频失效后只剩链接，找不到标题、封面和说明
- 希望把收藏数据长期保存在本地，并能导出、备份和恢复

## 当前功能

- 本地优先：数据默认保存在浏览器本地
- 收藏夹管理：新建、重命名、排序、描述、删除、回收站恢复
- 自定义标签：支持管理标签，并在收藏时基于已有标签进行补全/选择
- 检索能力：
  - 全局关键词检索
  - 按标题、UP 主、简介、B 站标签、自定义标签筛选
  - 按日期范围筛选
- 批量操作：批量移动、复制、删除视频
- 悬浮收藏面板：在 B 站视频页直接收藏到本地收藏夹
- 快捷键收藏：默认 `Ctrl+Alt+1` 打开收藏面板，`Enter` 确认
- 已收藏反馈：再次收藏同一视频时，可直接看到它已勾选在哪些收藏夹中
- 取消勾选即移除：在收藏面板中取消勾选某收藏夹并确认，即可移除对应收藏关系
- 记忆上一次收藏夹：方便高频重复整理
- 收藏夹合集播放：可从当前收藏夹批量打开可播放视频
- 关注 UP：批量导入当前账号关注列表，并在插件内快速搜索/跳转 UP 空间
- 同步导入：
  - 支持从 B 站选择收藏夹同步到本地
  - 按当前列表顺序逐个收藏夹同步，而不是一次性并发全部请求
  - 支持断点续传、自动冷却、降低风控概率的节流策略
- 收藏动作监听：可监听 B 站收藏动作，并自动对账到本地
- 可查看已收藏视频的卡片详情，一定程度上杜绝视频失效后无法知悉的“怅然若失”
- 失效视频补全：可对失效视频尝试补全标题、封面、简介等信息
- 导入/导出：支持 `JSON` / `CSV`
- WebDAV 备份（使用前建议创建同名目录）：支持配置、测试连通性、上传备份、下载备份与恢复
- 管理中心体验：支持深色/浅色主题与中英文切换

## 安装使用

### 方式一：普通用户

1. 下载：

- (1).前往浏览器扩展商店下载（商店版本一般落后于 Release）：
- Edge：https://microsoftedge.microsoft.com/addons/detail/bilishelf-manager/
- Firefox：https://addons.mozilla.org/zh-CN/firefox/addon/bilishelf/

- (2).前往仓库 `Releases` 页面，下载对应浏览器的插件压缩包：
- `https://github.com/TLRKFXE/BiliShelf/releases`
- 解压下载的插件包，打开浏览器扩展管理页面并安装：
- Chromium 内核浏览器（Chrome / Edge / Brave / Arc 等）：开启开发者模式后，选择“加载已解压的扩展程序”，指向插件目录。
- Firefox：通过“调试附加组件”或“从文件安装附加组件”安装对应包。

2. 打开任意 B 站视频页，用悬浮收藏面板或快捷键开始收藏
3. 点击浏览器工具栏中的插件入口，打开管理中心

### 方式二：开发 / 构建

安装根目录依赖：

```bash
pnpm install
```

安装前端与插件依赖：

```bash
pnpm --dir frontend install
pnpm --dir extension install
```

启动插件开发模式：

```bash
pnpm ext:dev
```

构建三端插件：

```bash
pnpm ext:build:all
```

打包三端压缩包：

```bash
pnpm ext:zip:all
```

## ⚠️ 注意事项

- 同步导入仍会受到 B 站风控影响，无法承诺 100% 无触发
- 如果出现 `412`：
  - 先确认没有重复打开多个管理页、多个浏览器 profile，或旧扩展实例残留
  - 关闭相关页面后稍等，再重新打开插件重试
- WebDAV 若测试连通性时报 `409`：
  - 通常说明目标路径已存在但不是期望的目录结构，或服务端不允许当前写入方式
  - 请检查填写的是目录地址而不是文件地址，并确认账号对该路径有创建/删除测试文件的权限
- 建议定期导出备份，避免本地数据意外丢失
- 纯本地存储，不上传个人收藏数据，隐私风险低。
- 如有问题请及时提交 `Issue`，也欢迎贡献 `PR`。

- 构建与工具链：pnpm、tsup、tsx、Vite、WXT
